3.  **Проект:** Это ОДНОСТРАНИЧНЫЙ сайт (`single_page=true`). Это означает, что все внутренние ссылки должны быть якорными и вести на разделы внутри этого же документа.

4.  **Точность:** Ты не придумываешь факты, статистику, цитаты или данные, а берешь их из интернета из соответствующих сайтов.

# Пошаговый план действий



### Шаг 1: Определение актуальной даты и создание YAML Frontmatter

1.  **Определи сегодняшнюю дату.** Эту дату ты будешь использовать для `lastmod` и для замены всех плейсхолдеров в тексте.

2.  Сформируй блок YAML в начале файла со следующими полями:

    - `title`: Используй один из H1-вариантов из ТЗ.

    - `description`: Напиши meta-description длиной 150-160 символов, основываясь на содержании статьи.

    - `lang`: Возьми из блока `<inputs>`.

    - `canonical`: Сформируй на основе `domain` из `<inputs>`.

    - `lastmod`: Укажи сегодняшнюю дату в формате YYYY-MM-DD.

    - `logo_concept`: Возьми из ТЗ.

    - `navigation`: **На основе "Плана навигации" из ТЗ, создай массив `navigation`.** Каждый элемент должен иметь ключи `text` и `anchor`.

    - `header_element`: **Это критически важно!** Проанализируй раздел "Элемент взаимодействия в шапке" в ТЗ и создай соответствующий YAML-объект:

        - Если в ТЗ указано **"Нет"**: `header_element: { type: 'none' }`

        - Если в ТЗ указано **"Строка поиска"**: `header_element: { type: 'search' }`

        - Если в ТЗ указано **"Кнопка действия (CTA)"**: `header_element: { type: 'cta', text: 'Текст кнопки из ТЗ', link: '#якорь-из-тз' }` (ссылку возьми из плана навигации, если она релевантна).

        - Если в ТЗ указано **"Контактная информация"**: `header_element: { type: 'contact', email: 'email@из.тз', phone: '+46...' }`

    - `footer_variant`: **Это критически важно!** Проанализируй раздел "Структура подвала (Footer)" в ТЗ и создай соответствующий YAML-объект.

        - **Пример:** Если в ТЗ указан "ID 1: Минималистичный" с определенным составом, YAML должен выглядеть так:

          `footer_variant: { type: 'minimalistic', elements: ['navigation_short', 'copyright', 'age_gate'] }`

        - Если указан "ID 2: Корпоративный":

          `footer_variant: { type: 'corporate', elements: ['logo', 'contact', 'socials', 'copyright', 'age_gate'] }`

        - Если указан "ID 3: Навигационный":

          `footer_variant: { type: 'navigation', elements: ['logo', 'navigation_full', 'search', 'copyright', 'age_gate'] }`

        - Если указан "ID 4: Комплексный":

          `footer_variant: { type: 'comprehensive', elements: ['logo', 'navigation_short', 'socials', 'contact', 'copyright', 'age_gate'] }`



### Шаг 2: Написание основного текста статьи

1.  **Замена плейсхолдеров:** Внимательно просмотри **весь текст из ТЗ** и найди все вхождения `[текущий год]` и `[текущий месяц]`. Замени их на актуальные значения.

2.  **Структура:** Строго следуй плану (H1, H2, H3...) из ТЗ.

3.  **Содержание:** Раскрой каждый пункт плана, опираясь на краткое содержание из ТЗ. Используй свои знания и данные из интернета для написания экспертного текста.

4.  **Ключевые сущности (LSI):** Естественно интегрируй в текст ключевые сущности, перечисленные в ТЗ.

5.  **Форматирование:** Используй Markdown для таблиц и списков, как указано в ТЗ.

6.  **Вставка плейсхолдеров для изображений:** Найди в ТЗ таблицу с планом изображений и вставь плейсхолдеры в формате `![Alt текст](placeholder "figure:ID;slug:slug")` в соответствующие разделы текста.

7.  **Внутренние ссылки:** Все внутренние ссылки должны быть якорными, вида `[текст ссылки](#anchor-id)`. `anchor-id` формируется из текста заголовка (латиница, нижний регистр, пробелы заменены на дефис).

8.  **Обработка плейсхолдеров в контенте:** Если в таблицах или тексте ТЗ ты встречаешь плейсхолдеры (например, "Alternativ/Casino"), замени их на реальные, релевантные данные, найденные в интернете.



### Шаг 3: Обеспечение комплаенса и E-E-A-T

1.  Размести все обязательные юридические оговорки (18+, Responsible Gambling) в конце статьи, перед разделителем `---`.

2.  Убедись, что упоминания брендов и лицензий соответствуют требованиям ТЗ.

</instructions>
