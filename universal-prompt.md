Мы работаем над задачей из TODO.md.
Текущая задача:
"""
[ВСТАВИТЬ ТЕКСТ ЗАДАЧИ ИЗ TODO.MD ЗДЕСЬ, например: "Sprint 1: Создать интерфейс Publisher и реализацию LocalPublisher"]
"""

Твоя цель — реализовать эту задачу изолированно, безопасно и с возможностью проверки.

## 🛠️ Protocol (Strictly Follow Steps)

### Step 1: Impact Analysis & Plan
Сначала проанализируй:
1. Какие файлы нужно создать?
2. Какие существующие файлы нужно изменить? (Внимание: Не удаляй существующий код, если это не оговорено явно. Используй расширение функционала).
3. Какие новые зависимости (imports) понадобятся?
Напиши краткий план изменений перед кодом.

### Step 2: Test/Verification First (TDD style)
Перед написанием основного кода, напиши:
- Либо **Unit Test** (`_test.go`), который проверяет новую логику.
- Либо **Verification Script** (например, `cmd/verify_task/main.go` или `curl` команду), который я смогу запустить локально, чтобы убедиться, что код работает.
*Критерий:* Тест должен проверять как успешный сценарий, так и обработку ошибок.

### Step 3: Implementation
Напиши полный код реализации.
- Используй Go idioms (context handling, error wrapping).
- Если меняешь БД — приложи SQL миграцию.
- Если меняешь API — приложи обновленный handler.
- Обязательно добавь комментарии к экспортируемым методам.

### Step 4: Self-Check Checklist
В конце ответа выполни обязательно этот чек-лист:
- [ ] Не сломана ли компиляция?
- [ ] Обработаны ли ошибки?
- [ ] Соответствует ли интерфейсам из TODO?

Жду от тебя выполнения по шагам 1-4.
