# 🗺 Полный план разработки: SiteGENAI 2.0 (UX/UI Рефакторинг)

## Этап 1: Фундамент и Навигация (Глобальный Layout)
*Цель: Избавиться от монолитной шапки, внедрить профессиональный SaaS-интерфейс и разделить логику по ролям.*

1. **Реорганизация Route Groups:** Разделение приложения на `(auth)` (публичные страницы входа/регистрации) и `(app)` (внутренняя закрытая часть).
2. **Создание `DashboardLayout`:**
   * **Левый сайдбар (Sidebar):** Главное навигационное меню. Сворачиваемое.
   * **Умный роутинг (RBAC):** Копирайтер (`editor`) видит в сайдбаре только "Мои сайты" и "Профиль". Админ/Владелец видят разделы "Управление", "Очереди", "Мониторинг" и "Админка".
   * **Верхняя панель (Topbar):** Динамические хлебные крошки (Breadcrumbs) для навигации вглубь (`Проекты / Мой проект / example.com`).
3. **Глобальные алерты:** Внедрение желтого баннера *"⚠️ API-ключ Gemini не настроен"*, который висит везде, пока Владелец не добавит ключ (чтобы генерации не падали молча).

## Этап 2: Лицо продукта (Проекты и Домены)
*Цель: Сделать управление сетками сайтов визуально приятным и избавить списки от технических инпутов.*

1. **Редизайн `/projects`:** 
   * Замена скучного списка на красивую сетку карточек (Grid).
   * Вынос формы создания проекта в модальное окно (Dialog).
2. **Редизайн `/projects/[id]` (Карточка проекта):**
   * Превращение страницы в рабочее пространство с табами (для Админа: Сайты, SEO, Расписания, Настройки). Для редактора — просто список сайтов.
   * Вынос инпутов импорта и добавления домена в отдельные модальные окна.
3. **Редизайн `/domains/[id]` (Карточка сайта):**
   * Акцент на главной кнопке: **«📝 Открыть редактор контента»**.
   * Создание компонента **Pipeline Visualizer**: визуальная шкала шагов генерации сайта вместо текстовой таблицы (Сбор -> Контент -> HTML -> Готово).
   * Вынос настроек (Keyword, Anchor, Server) в аккуратный боковой блок или модалку.

## Этап 3: The Ops Center (Очереди и Расписания)
*Цель: Превратить сложные базы данных в удобный пульт управления с массовыми действиями.*

1. **Очереди генерации и ссылок (`/queue`):**
   * **Кликабельные дашборды:** Фильтры превращаются в виджеты над таблицей (например, клик по красной плашке `[Ошибок: 12]` мгновенно фильтрует таблицу).
   * **Массовые действия:** Добавление чекбоксов в таблицу для выделения нескольких зависших задач и нажатия кнопки "🔄 Перезапустить выделенное".
   * **Side Drawers (Выезжающие шторки):** Удаление отдельной страницы `queue/[id]`. При клике на задачу, справа выезжает шторка с логами и кнопкой перезапуска. Контекст списка не теряется!
   * **Человечные ошибки:** Парсинг популярных ошибок (например, `context_too_large`) в понятные UI-бейджи.
2. **Умные Расписания (`/schedules`):**
   * Замена сырых полей cron на **Sentence Builder** (конструктор предложений: *"Запускать[каждый день] в [10:00]"*).
   * Добавление прогноза: *"Следующий запуск: сегодня в 14:00"*.
   * Карточный вид (как будильники в телефоне) с большими тумблерами Вкл/Выкл.

## Этап 4: Админка, Промпты и Мониторинг
*Цель: Дать админам мощные, но красивые инструменты аналитики и настройки AI.*

1. **Prompt Studio (`/admin`):**
   * Редизайн управления системными промптами.
   * Использование Monaco Editor для промптов с **подсветкой синтаксиса переменных** (`{{ keyword }}`).
   * Панель-шпаргалка со списком доступных переменных для каждого этапа генерации.
2. **Мониторинг индексации (`/monitoring/indexing`):**
   * Визуализация "Heatmap" (как график активности в GitHub), чтобы с одного взгляда видеть динамику индексации.
3. **Биллинг LLM (`/monitoring/llm-usage`):**
   * Добавление простых графиков (Bar chart) расходов по дням и Pie chart расходов по моделям (Gemini Pro vs Flash) над существующей таблицей.

## Этап 5: Магия AI-Редактора (Notion-style)
*Цель: Скрыть HTML-код от копирайтера, заставив AI быть мостом между текстом и дизайном сайта.*

1. **Тумблер режимов:** В шапке редактора делаем переключатель: `[ 📝 Режим Контента ] / [ 👩‍💻 Код ]`.
2. **Левое меню (Сайдбар):**
   * Парсинг файлового дерева: скрываем `.html` и `.css`, показываем иконки страниц ("Главная", "Контакты").
3. **Визуальный редактор (Novel.sh / TipTap):**
   * Интеграция блочного редактора. Чистый белый лист для написания текста, таблиц и вставки кнопок.
   * SEO-панель справа (Title, Description, ЧПУ), которая сама инжектит мета-теги в `head`.
4. **AI-Компилятор:**
   * Подключение кнопки "Опубликовать" к вашему текущему процессу `ai-create-page`.
   * Копирайтер пишет абстрактный текст -> AI получает контекст сайта (`style.css`) -> AI верстает страницу -> Пользователь видит красивое превью в iframe и жмет "Применить".
5. **Генерация картинок:** Интеграция генерации изображений прямо в интерфейс блочного редактора (как slash-команда `/image`).
